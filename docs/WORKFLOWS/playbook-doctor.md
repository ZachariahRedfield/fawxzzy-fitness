# Playbook Doctor Workflow

## Purpose
`playbook:doctor` validates whether a repository is correctly wired to consume the central Playbook engine and prints immediate, actionable fixes.

## Run

```bash
npm run playbook:doctor
npm run playbook:doctor -- --cwd=/absolute/path/to/consumer-repo
npm run playbook:doctor -- --cwd=. --strict
```

## Checks performed
- Node runtime availability.
- Playbook engine scripts are present and runnable.
- `docs/PLAYBOOK_NOTES.md` exists.
- `docs/playbook-status.json` exists (or can be generated by running `npm run playbook`).
- `docs/playbook-trend.json` exists (or can be generated by running `npm run playbook`).
- `tools/playbook/config.json` exists.
- Pre-commit hook is installed and points to `npm run playbook` (non-blocking warning if missing).
- Required package scripts exist: `playbook`, `playbook:status:ci`.
- CI workflow guidance exists under `.github/workflows`.

## Exit behavior
- Exit `0` when checks are all `OK`, or only `WARN` checks are present.
- Exit `1` when any `FAIL` check exists.
- Exit `1` in `--strict` mode when `WARN` checks exist.

## Common fixes
- Install/update scripts, hook, and config:

```bash
node ./Playbook/tools/install/cli.mjs
```

- Generate status/trend files:

```bash
npm run playbook -- --cwd=.
```

- Add CI dashboard wiring from:
  - `Playbook/tools/install/CI_SNIPPET.md`

## Migration hint support
If the doctor detects legacy repo-local files in `scripts/playbook/*`, it prints migration hints to move scripts to central Playbook engine commands and safely remove legacy scripts after a green CI cycle.
